DEPLOYMENT GUIDE

四步完成上線。

準備一份 Google 試算表、一個 LINE Login Channel，以及可發布 GitHub Pages 的儲存庫。整套系統不需要安裝套件。

01

建立 Google Sheet 與 Apps Script

建立 Google 試算表並複製網址中 /d/ 與 /edit 之間的 Spreadsheet ID，再從「擴充功能 → Apps Script」開啟編輯器。

將 gas/Code.gs 貼入程式，並在專案設定開啟資訊清單檔案後，以 gas/appsscript.json 取代內容。

指令碼屬性

LINE_CHANNEL_ID
LINE Login Channel ID，不是 LIFF ID
SPREADSHEET_ID
剛才複製的 Google Sheet ID
ALLOWED_ORIGINS
前端 origin，例如 https://example.github.io
SHEET_NAME
選填，預設 Members
MAX_VERIFY_REQUESTS_PER_MINUTE
選填，LINE 驗證每分鐘上限，預設 120

ALLOWED_ORIGINS 不可填 *，也不要包含 GitHub Pages 的子目錄路徑。

02

初始化並部署 GAS

在 Apps Script 編輯器選擇 setup 執行一次並完成授權。程式會建立 Members 工作表與固定欄位。

選擇「部署 → 新增部署作業 → 網頁應用程式」，執行身分選「我」，存取權選「任何人」。複製結尾為 /exec 的正式網址；不要使用只供編輯者測試的 /dev。

curl -L "你的_GAS_EXEC_網址?action=health"

看到 "ok":true 即代表公開部署可連線。之後每次修改 GAS，都要在管理部署作業中建立新版本。

03

建立 LINE Login 與 LIFF

到 LINE Developers Console 建立 LINE Login Channel，並在 LIFF 分頁新增 App。

  • Endpoint URL：本會員頁部署後的完整 HTTPS 網址
  • Size：建議 Full
  • Scope：必要的 openid、profile，以及選用的 email

儲存後複製 LIFF ID。正式使用者應從 https://liff.line.me/{LIFF_ID} 開啟會員中心。

04

填入前端設定並發布

編輯 config.json，填入剛取得的 LIFF ID、GAS /exec 網址與品牌名稱。

{
  "LIFF_ID": "1234567890-AbCdEfGh",
  "GAS_WEB_APP_URL": "https://script.google.com/macros/s/.../exec",
  "BRAND_NAME": "你的品牌名稱"
}

提交到 GitHub Pages 發布分支後，用 LIFF URL 在 LINE 手機 App 與外部瀏覽器各測試一次。更完整的檢查與疑難排解請看 專案 README。
